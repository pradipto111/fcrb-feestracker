DATABASE MODELS (Prisma Schema)
================================
Located in: backend/prisma/schema.prisma

1. Center
   - Core fields: id, name, shortName, addressLine, locality, city, state, postalCode
   - Map fields: latitude, longitude, googleMapsUrl
   - UI fields: isActive, displayOrder
   - Legacy fields: location, address (for backward compatibility)
   - Relations: students, coachLinks, payments, sessions, fixtures, posts, votes, studentStats

2. Coach
   - Fields: id, fullName, email, passwordHash, role, createdAt, updatedAt
   - Relations: centers, sessions, fixtures, videos

3. CoachCenter
   - Fields: id, coachId, centerId
   - Relations: coach, center

4. Student
   - Fields: id, fullName, dateOfBirth, phoneNumber, parentName, parentPhoneNumber
   - Auth: email, passwordHash
   - Centre: centerId
   - Fees: monthlyFeeAmount, paymentFrequency
   - Status: status (ACTIVE, INACTIVE, TRIAL)
   - Relations: center, payments, attendance, fixturePlayers, stats

5. Payment
   - Fields: id, studentId, centerId, amount, paymentDate, paymentMode
   - Reference: upiOrTxnReference, notes
   - Relations: student, center

6. Session
   - Fields: id, centerId, coachId, sessionDate, startTime, endTime, notes
   - Relations: center, coach, attendance, votes

7. Attendance
   - Fields: id, sessionId, studentId, status (PRESENT, ABSENT, EXCUSED), notes
   - Relations: session, student

8. Fixture
   - Fields: id, centerId, coachId, matchType, opponent, matchDate, matchTime, venue
   - Status: status (UPCOMING, ONGOING, COMPLETED, CANCELLED, POSTPONED)
   - Relations: center, coach, players

9. FixturePlayer
   - Fields: id, fixtureId, studentId, position, role, notes
   - Relations: fixture, student

10. Video
    - Fields: id, title, description, videoUrl, platform, category, thumbnailUrl
    - Relations: creator (coach)

11. Post
    - Fields: id, title, description, mediaType, mediaUrl, platform, centerId
    - Approval: approvalStatus, approvedById, approvedAt
    - Relations: center, comments

12. Comment
    - Fields: id, postId, content, createdByRole, createdById
    - Relations: post

13. Vote
    - Fields: id, sessionId, voterRole, voterId, votedForId, points, centerId
    - Relations: session, center

14. StudentStats
    - Fields: id, studentId, centerId, totalPoints, studentVotes, coachVotes
    - Streaks: currentStreak, longestStreak
    - Periods: weeklyPoints, monthlyPoints, lastVotedAt
    - Relations: student, center, badges

15. Badge
    - Fields: id, studentStatsId, badgeType, earnedAt
    - Relations: studentStats

16. WebsiteLead
    - Fields: id, source, playerName, playerDob, ageBracket, guardianName, phone, email
    - Centre: preferredCentre, programmeInterest, playingPosition, currentLevel
    - Status: status (NEW, CONTACTED, CONVERTED, LOST)
    - Relations: assignedTo, convertedPlayerId

17. Product
    - Fields: id, name, slug, description, images, price, currency, sizes, variants
    - Stock: stock, isActive
    - UI: category, tags, displayOrder
    - Relations: orderItems

18. Order
    - Fields: id, orderNumber, subtotal, shippingFee, total
    - Customer: customerName, phone, email, shippingAddress
    - Payment: status, paymentProvider, paymentReference, paymentData
    - Relations: items

19. OrderItem
    - Fields: id, orderId, productId, productName, variant, size, quantity
    - Pricing: unitPrice, totalPrice
    - Relations: order, product


API ENDPOINTS
=============
Backend Base URL: http://localhost:4000 (or configured VITE_API_URL)

AUTHENTICATION
  POST   /auth/login              # Login
  POST   /auth/register           # Register (if enabled)

PUBLIC ENDPOINTS (No Auth Required)
  GET    /centers/public          # Get active centres for homepage map
  GET    /shop/products           # Get active products
  GET    /shop/products/:slug     # Get product by slug
  POST   /leads                   # Create website lead

AUTHENTICATED ENDPOINTS (Auth Required)
  GET    /centers                 # Get centres (admin: all, coach: assigned)
  POST   /centers                 # Create centre (admin only)
  PUT    /centers/:id             # Update centre (admin only)
  DELETE /centers/:id             # Delete centre (admin only)
  GET    /centers/:id             # Get centre details (admin only)

  GET    /students                # Get students (admin/coach)
  POST   /students                # Create student (admin only)
  PUT    /students/:id            # Update student (admin only)
  DELETE /students/:id            # Delete student (admin only)
  GET    /students/:id            # Get student details

  GET    /attendance              # Get attendance records
  POST   /attendance              # Create attendance record
  PUT    /attendance/:id          # Update attendance record

  GET    /fixtures                # Get fixtures
  POST   /fixtures                # Create fixture
  PUT    /fixtures/:id            # Update fixture
  DELETE /fixtures/:id            # Delete fixture

  GET    /videos                  # Get videos/drills
  POST   /videos                  # Create video (admin/coach)
  PUT    /videos/:id              # Update video (admin/coach)
  DELETE /videos/:id              # Delete video (admin/coach)

  GET    /posts                   # Get posts
  POST   /posts                   # Create post
  PUT    /posts/:id               # Update post
  DELETE /posts/:id               # Delete post
  POST   /posts/:id/approve       # Approve post (admin/coach)

  GET    /leaderboard             # Get leaderboard
  POST   /votes                   # Submit vote

  GET    /admin/leads             # Get website leads (admin only)
  GET    /admin/leads/:id         # Get lead details (admin only)
  PUT    /admin/leads/:id         # Update lead (admin only)
  POST   /admin/leads/:id/convert # Convert lead to player (admin only)

  GET    /admin/merch             # Get all products (admin only)
  GET    /admin/merch/:id         # Get product details (admin only)
  POST   /admin/merch             # Create product (admin only)
  PUT    /admin/merch/:id         # Update product (admin only)
  DELETE /admin/merch/:id         # Delete product (admin only)
  PUT    /admin/merch/:id/toggle-active # Toggle product active status (admin only)

  POST   /shop/orders/create      # Create order (public, but requires cart)
  POST   /shop/orders/:orderId/verify # Verify payment (public)
  GET    /shop/orders/:orderNumber # Get order details (public)

  GET    /dashboard               # Get dashboard summary (admin/coach)
  GET    /student/dashboard       # Get student dashboard data
  GET    /student/profile         # Get student profile
  GET    /student/attendance      # Get student attendance

  GET    /system/date             # Get system date (admin only)
  PUT    /system/date             # Set system date (admin only)
